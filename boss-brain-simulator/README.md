# 🏛️ 五人决策小组 · Boss Brain Simulator

一个基于百炼 CLI (`bl pipeline`) 的多步骤决策分析工具。适用于项目立项、资源申请、商业化判断、跨部门协调、重大汇报、项目延期和职业决策等复杂场景。

## 决策者阵容

| 决策者 | 视角 | 核心问题 |
|--------|------|----------|
| 💰 巴菲特 | 资本与风险 | 这事值得投入吗？失败能承受吗？ |
| ⚡ 马斯克 | 第一性原理与效率 | 能不能去掉假设，找到更小更快的验证路径？ |
| 🏢 德鲁克 | 组织与顾客 | 为谁创造价值？谁对结果负责？ |
| 🔍 芒格 | 反向思考与决策偏差 | 怎样一定会失败？有什么认知盲区？ |
| 🎯 格鲁夫 | 执行与结果管理 | 当前最大瓶颈是什么？能落地吗？ |

## 五人决策规则

1. 五个人不能重复表达相同观点
2. 每个人必须从自己的决策领域提出判断
3. 不得为了制造冲突而虚构问题，但必须主动寻找合理的反对意见
4. 当五人意见一致时，芒格必须提出最强反方观点
5. 当方案过于复杂时，马斯克必须提出删减方案
6. 当价值或付费方不清晰时，德鲁克和巴菲特可以建议暂缓
7. 当结论正确但无法执行时，格鲁夫必须指出责任人、时间或验收机制缺失
8. 最终不能停留在「各有利弊」，必须形成推荐结论

## 三轮流程

1. **澄清**：只准提问，不准给建议，打在没说出口的假设上
2. **交锋**：必须有真实分歧，互相 @ 反驳
3. **收敛**：形成推荐结论 + 14 天 3 个动作 + 止损信号 + 验证指标

## 用法

```bash
cd /Users/oceg/agents/boss-brain-simulator

# 用 problem.json 里的内容跑
./run.sh

# 直接传问题
./run.sh "我想申请加两个人，但老板觉得现在够用"

# 指定输入文件
./run.sh --file ./problem.json

# 对话模式（适合追问）
./run.sh --chat
```

## 前置条件

- 已安装 `bailian-cli`：`npm install -g bailian-cli`
- 已登录：`bl auth login --console` 或 `bl auth login --api-key sk-xxx`

## 文件结构

```
boss-brain-simulator/
├── pipeline.yaml      # 3步链式流水线（澄清 → 交锋 → 收敛）
├── workflow.yaml      # 13步并行流水线（5位决策者各自独立发言）
├── system-prompt.md   # 单 prompt 对话版
├── problem.json       # 问题模板
├── run.sh             # 启动脚本（默认跑 pipeline.yaml）
└── README.md          # 本文件
```
